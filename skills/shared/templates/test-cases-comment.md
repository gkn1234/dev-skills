### ✅ 回归测试用例

#### 正常流程

**用例 1：使用有效凭证登录**
1. 访问 `/login`
2. 在用户名输入框输入 `user@example.com`
3. 在密码输入框输入有效密码
4. 点击「登录」按钮
5. 验证：跳转至 `/dashboard`，顶部导航栏显示用户名

- [ ] 通过

**用例 2：记住登录状态**
1. 访问 `/login`
2. 输入有效凭证，勾选「记住我」
3. 点击「登录」，登录成功
4. 关闭浏览器，重新打开
5. 访问 `/dashboard`
6. 验证：无需再次登录，直接进入页面

- [ ] 通过

**用例 3：退出登录**
1. 已登录状态访问 `/dashboard`
2. 点击右上角「退出登录」按钮
3. 验证：跳转至 `/login`，localStorage 中 token 已清除

- [ ] 通过

#### 边界场景

**用例 4：用户名为空提交**
1. 访问 `/login`
2. 用户名留空，输入密码
3. 点击「登录」按钮
4. 验证：用户名输入框下方显示红色提示"请输入用户名"

- [ ] 通过

**用例 5：密码最小长度**
1. 访问 `/login`
2. 输入用户名
3. 输入 6 位密码（系统最小长度）
4. 点击「登录」按钮
5. 验证：密码校验通过，表单正常提交

- [ ] 通过

#### 异常场景

**用例 6：密码错误**
1. 访问 `/login`
2. 输入正确用户名但错误密码
3. 点击「登录」按钮
4. 验证：显示 toast "用户名或密码错误"，密码框清空

- [ ] 通过

**用例 7：账户锁定**
1. 访问 `/login`
2. 连续 5 次输入错误密码
3. 验证：显示"账户已锁定，请 15 分钟后重试"，登录按钮禁用

- [ ] 通过

**用例 8：网络异常**
1. 访问 `/login`
2. 断开网络
3. 点击「登录」按钮
4. 验证：显示 toast "网络异常，请检查连接"

- [ ] 通过
